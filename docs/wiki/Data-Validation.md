# 数据检查

检查页把每一项明确标为“错误”“警告”或“通过”，并按这个优先级排序。

## 结果含义

- 错误：数据缺失、损坏或语义不可用。回放可用于定位问题，但正常导出会在 UI 和 Rust 后端同时被阻止。
- 警告：数据仍可读取，但存在缺帧、数量不一致或时间间隔异常等质量风险。确认警告后可以导出。
- 通过：本次检查覆盖范围内没有发现问题。

## JPEG 抽检范围

交互检查不会逐张解码全部图像。每个非空视频流按排序后帧序列固定抽取 `1% / 25% / 50% / 73% / 99%` 五个位置；不足五个唯一位置时自动去重。

目录、文件名、帧 ID、缺帧、状态 JSON 和时间戳仍然全量检查。检查页会同时显示流总帧数和实际解码帧数。抽检通过只代表被抽中的 JPEG 可以解码，不能证明未抽中的所有 JPEG 都正常。

正式压力和发布验收使用全量 JPEG 解码，不能用交互抽检报告替代。

## 报告

出现警告或错误时，应用会在后台自动生成本地报告；全部通过时不会生成无意义报告。相同 episode 路径、数据指纹和报告版本会去重。

检查页的“导出报告”用于把当前机器可读报告保存到用户选择的位置。报告包含检查模式、抽检百分位、流统计、issue code、可定位帧号和后台报告路径。

后台报告只写应用 local-data，不会上传网络，也不会写入 SD 卡或导入副本。
